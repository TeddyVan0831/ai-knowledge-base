## 摘要

反思模式让智能体在完成任务后对自己生成的结果进行自我评估和改进——通过"生成→反思→修正"的循环，显著提升输出质量。

## 详情

### 模式概述

反思模式的核心是**自我校验与迭代改进**——智能体先生成初步结果，然后切换到"评审者"角色，对结果进行评估并提出改进建议，最后根据反馈进行修正 。

**基本循环**：
```
生成初稿 → 反思评估 → 提出改进 → 修改结果 → (可选) 再次反思 → 输出终稿
```

### 实践应用与场景

- **代码生成**：生成代码 → 自检逻辑错误 → 修复 Bug
- **文章写作**：起草文章 → 评审流畅度和逻辑 → 润色修改
- **翻译任务**：初步翻译 → 反向翻译验证 → 调整偏差

### 关键要点

- 反思模式的关键是**提示词设计**——"评审者"和"生成者"需要不同的系统提示
- 反思循环次数需要控制（通常 1-3 次），避免无限循环
- 反思可以显著提升**质量敏感型任务**的输出，但会增加**延迟和成本**

### 与 Agent 反思的区别

- 反思模式是单步的自我评估
- Agent 反思（如 ReAct Agent 的反思组件）是整个 Agent 架构中的持续反思能力
- 前者更轻量，后者更系统化
