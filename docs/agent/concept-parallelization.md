## 摘要

并行化模式将独立子任务同时发送给多个 LLM 实例执行，通过并发处理显著缩短总耗时，适用于可分割且子任务间无依赖的大规模处理场景。

## 详情

### 模式概述

并行化模式的核心是**分片并发执行**——将一个大任务拆分为多个独立子任务，同时发送给多个 LLM 实例处理，最后汇总结果 。

**两种并行方式**：
- **分块并行**：将输入数据分块，每块独立处理（如批量文档摘要）
- **投票并行**：同一问题发送给多个模型，通过投票或取最佳结果（如多模型评审）

### 实践应用与场景

- **批量文档处理**：同时处理 100 份文档的摘要
- **多模型评审**：同一答案发送给 3 个模型独立评分，取平均分
- **大规模数据分析**：将数据集分片，并行分析后合并

### 关键要点

- 并行化适合**子任务独立、无状态依赖**的场景
- 需要权衡**成本**（多个 API 调用）与**速度**（缩短总耗时）
- 汇总结果时需要考虑**结果一致性**（不同模型可能有不同输出格式）

### 与路由的区别

- 路由是"选一"：根据输入类别选择一条处理路径
- 并行是"多选"：同时执行多条路径，合并结果
