# Agent 自主规划 — 五大架构模式

> **类型**: concept
> **创建时间**: 2026-05-26
> **最后更新**: 2026-05-26
> **来源**: 

## 摘要

AI Agent 是通过 LLM 能够**自主理解、规划、执行复杂任务**的系统 。Agent 的核心模块包括**规划（Planning）、记忆（Memory）、工具使用（Tool Use）** 。五种架构不是互斥关系，而是**能力递进关系**——从最基础的即时响应，到最高级的持续自进化 。

## 详情

### AI Agent 核心模块

| 模块                 | 说明                             |
| ------------------ | ------------------------------ |
| **规划（Planning）**   | 将复杂任务分解为较小的、可管理的子目标            |
| **记忆（Memory）**     | 短期记忆（上下文学习）+ 长期记忆（外部载体储存与快速检索） |
| **工具使用（Tool Use）** | 调用外部 API，获取额外信息                |


### Agent 与自动化的区别

- **自动化**：固定路径，预先编程好每一步 
- **自主化**：自主决策，根据环境反馈动态调整 

### 五种架构全景图

```
能力复杂度递进 ──────────────────────────────────────────────────→

┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐   ┌──────────┐
│  反应式   │ → │  深思熟虑 │ → │  ReAct   │ → │  PER     │ → │  自进化   │
│ Reactive │   │Deliberative│ │          │   │(Plan+Reflect)│ │(PER+Memory+Learn)│
└──────────┘   └──────────┘   └──────────┘   └──────────┘   └──────────┘
  即时响应        内部建模         思考-行动交替      规划+反思闭环      持续学习进化
  无记忆         长期目标         工具调用          自我修正         经验积累
```


### 五种架构核心对比

| 维度          | 反应式（Reactive） | 深思熟虑（Deliberative） | ReAct                             | PER（规划+反思）                        | 自进化（PER+记忆+学习）      |
| ----------- | ------------- | ------------------ | --------------------------------- | --------------------------------- | ------------------- |
| **核心模式**    | 感知→决策→执行      | 感知→建模→推理→决策→执行     | Thought → Action → Observation 循环 | Plan → Execute → Reflect → Refine | PER 循环 + 记忆库 + 学习反馈 |
| **是否有内部模型** | ❌             | ✅ 世界模型             | ❌                                 | ✅ 反思报告                            | ✅ 经验记忆库 + 模型参数更新    |
| **是否能自我修正** | ❌             | ✅ 计划内修正            | ✅ 通过 Observation 修正               | ✅ 通过 Reflect 阶段主动反思               | ✅ 持续学习，永久改进         |
| **记忆能力**    | 无             | 任务内上下文             | 对话上下文                             | 反思记录                              | 长期记忆库 + 经验复用        |
| **学习进化能力**  | ❌             | ❌                  | ❌                                 | ❌（单次任务内修正）                        | ✅ 跨任务持续进化           |
| **推理成本**    | 低             | 高                  | 中                                 | 高                                 | 非常高                 |
| **典型延迟**    | 毫秒~秒级         | 分钟级                | 秒~分钟级                             | 分钟级                               | 分钟~小时级              |
| **最佳任务类型**  | 简单、规则明确、需即时响应 | 复杂、多步、需战略规划        | 需要多轮工具调用的复杂问答                     | 需要高质量输出和自我修正的任务                   | 需要持续优化和专业能力积累的领域    |


### 各架构详解

#### 1. 反应式 Agent（Reactive）
```
感知 → 决策（LLM/规则） → 执行 → 观察反馈 → 循环
```

- **优势**：速度快（毫秒级响应）、简单可靠 
- **局限**：缺乏适应性、短视性 
- **典型应用**：扫地机器人避障、无人机紧急悬停、游戏 NPC 即时反应、工业控制传感器告警 

#### 2. 深思熟虑式 Agent（Deliberative）
```
感知 → 建模（更新内部世界状态） → 推理（生成候选计划并模拟） → 决策（选择最优方案） → 执行
```

- **优势**：能处理多步复杂任务、优化长期目标、适应动态变化环境 
- **典型应用**：路径规划智能体、物流调度、投资决策 

#### 3. ReAct 架构（Reasoning + Acting）
由 Yao et al.（2022）提出 ：
```
Thought → Action → Observation → Thought → Action → Observation → ... → Final Answer
```

- **优势**：推理过程透明、灵活适应未知任务、天然支持工具调用、实现简单 
- **局限**：可能陷入循环思考、Token 消耗大、缺乏长期记忆 

#### 4. PER 架构（Plan → Execute → Reflect）
四阶段闭环 ：
- **Plan（规划）**：分解任务、制定策略、选择工具、设定评估标准
- **Execute（执行）**：按计划逐步执行，调用工具，收集信息
- **Reflect（反思）**：对比目标评估结果，检查逻辑一致性、完整性、准确性
- **Refine（修正）**：根据反思结果修正计划或重新执行

与 ReAct 的区别：ReAct 是"每一步行动后立即观察（微循环）"，PER 是"整个任务完成后进行整体反思（宏循环）" 。

#### 5. 自进化架构（PER + 记忆 + 学习）
Agent 架构的最高形态，Agent 不仅能完成任务和自我修正，还能**从经验中学习**，将反思结果沉淀为长期记忆 。

**五大核心组件** ：
| 组件           | 功能说明                                     | 技术实现                     |
| ------------ | ---------------------------------------- | ------------------------ |
| **PER 循环引擎** | Plan → Execute → Reflect → Refine 核心推理循环 | LLM + Prompt Engineering |
| **长期记忆库**    | 存储成功经验、失败教训、最佳实践、用户偏好等                   | 分层文件系统、向量数据库（可选）、知识图谱    |
| **记忆检索**     | 根据当前任务上下文检索相关历史经验                        | 语义相似度搜索 + 关键词匹配          |
| **学习更新机制**   | 从反思结果中提炼可复用的策略，更新记忆库和决策模型                | 经验压缩、策略归纳、强化学习           |
| **反馈强化**     | 接收用户/环境反馈，调整行为策略                         | RLHF、在线学习、A/B 测试         |

**代表产品**：Hermes Agent（Nous Research，2026 年 2 月开源），是首个完整实现 PER + 记忆 + 学习三要素闭环的开源 Agent 框架 。

### 选型决策树

```
你的任务需求是什么？
│
├── 需要即时响应、规则明确？→ 反应式（Reactive）
├── 需要多步规划和内部建模？→ 深思熟虑（Deliberative）
├── 需要多轮工具调用和信息检索？→ ReAct
├── 需要高质量输出和自我修正？→ PER（规划+反思）
└── 需要持续学习和能力积累？→ 自进化（PER + 记忆 + 学习）
```


### 构建 Agent 的三大核心思想

1. **不要为所有任务构建 Agent**：Agent 适合处理**复杂、模糊且高价值**的任务，而非所有场景 
2. **保持简洁**：Agent 的核心组件只有三个——环境、工具集、系统提示 
3. **像 Agent 一样思考**：Agent 仅基于有限上下文（10-20k Token）做决策，需模拟 Agent 的视角来发现设计缺陷
