# Function Calling — 模型的"手指"

> **类型**: concept
> **创建时间**: 2026-05-26
> **最后更新**: 2026-05-26
> **来源**: 

## 摘要

Function Calling 是大模型的核心能力之一，让模型能够**调用预设的外部函数**来完成任务。模型将用户的自然语言请求转化为**结构化的函数调用**（函数名 + 参数），是 Agent 架构中最细粒度的"原子化能力" 。

## 详情

### 什么是 Function Calling

Function Calling 让 LLM 能够**调用预设的外部函数**来完成任务 。本质上，模型将自然语言请求转化为**结构化的函数调用**，包括函数名和参数 。

### 三大作用

| 作用         | 说明                                                            | 示例                                                                    |
| ---------- | ------------------------------------------------------------- | --------------------------------------------------------------------- |
| **扩展模型能力** | 模型本身无法直接操作外部系统，通过调用函数可以：获取实时数据（天气、股价）、执行复杂计算、操作外部系统（发邮件、控制设备） | 查询实时天气、执行数学运算  |
| **结构化输出**  | 将自然语言转化为结构化参数传递给函数                                            | 用户说"明天北京天气如何？" → 模型调用 `get_weather(location="北京", date="2025-05-06")`  |
| **动态决策流程** | 模型根据上下文自主决定是否调用函数、何时调用，甚至链式调用多个函数                             | 先查天气 → 再推荐穿搭  |

### 工作流程

```
用户输入自然语言问题
    ↓
LLM 解析意图，自动生成函数调用（含参数）
    ↓
系统执行函数，返回结果
    ↓
LLM 将结果转化为自然语言回答用户
```


### Function Calling vs Skill vs MCP

| **维度**     | **Function Calling** | **Skill（技能/插件）**     | **MCP（模型上下文协议）**       |
| ---------- | -------------------- | -------------------- | ---------------------- |
| **本质**     | **原子化能力**            | **产品化封装**            | **标准化接口**              |
| **定位**     | 模型解析意图并输出参数的"手"      | 面向用户/业务的功能集合"工具箱"    | 解决模型与工具连接的"通用插座"       |
| **粒度**     | 最细（单个函数定义）           | 中等（组合多个函数与逻辑）        | 宏观（定义连接的标准与生态）         |
| **开发者关注点** | JSON Schema、参数准确性    | 业务流程、Prompt、UI 展示    | 协议适配、跨模型兼容性            |
| **典型平台**   | OpenAI API, Qwen API | Dify, Coze, GPTs     | Claude Desktop, Cursor |
| **复用性**    | 低（通常为特定任务编写）         | 中（可在相同平台的 Agent 中复用） | **极高**（一次开发，多平台/模型使用）  |


### 形象比喻

- **Function Calling** 是"手指"：模型学会了如何按按钮 
- **Skill** 是"瑞士军刀"：为了剪指甲，你把剪刀、锉刀封装在一起，并贴上说明书 
- **MCP** 是"USB-C 接口"：无论你是哪种工具，只要插在这个标准接口上，所有支持该协议的模型都能直接用 

### 实战：门票助手

搭建门票数据查询助手，用户使用自然语言提问，系统自动完成 SQL 查询并返回结果 。

**四步法搭建** ：
1. **系统初始化**：设置 system prompt，描述门票表结构，注册 SQL 查询工具（`exc_sql`）
2. **助手实例化**：使用 Qwen-Agent 的 `Assistant` 类，加载 LLM 配置、system prompt 和 `function_list`
3. **设置交互模式**：选择 WebUI 模式，用户通过网页输入问题
4. **Function Call 机制运作**：用户输入自然语言 → LLM 解析意图并自动生成 SQL → `exc_sql` 工具被自动调用 → 结果展示给用户 

### 选型建议

- **如果你在开发一个特定的小功能**（如查实时股价）：直接用 **Function Calling** 
- **如果你在构建一个复杂的 Agent 平台**（如企业内部助手）：将功能封装为 **Skill** 
- **如果你想让你开发的工具被全世界的 AI 开发者使用**（如 Cursor 插件）：必须适配 **MCP**
