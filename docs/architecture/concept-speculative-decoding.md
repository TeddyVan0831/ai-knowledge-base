# 投机解码（Speculative Decoding）

> **类型**: concept
> **创建时间**: 2026-05-27
> **最后更新**: 2026-05-27
> **来源**: 

## 摘要

投机解码是 LLM 推理加速的核心算法——用小模型（草稿模型）快速生成 γ 个候选 token，大模型（目标模型）一次性验证，通过接受-拒绝机制保证输出分布完全不变，实现 1.7-3.5x 加速 。

## 详情

### 核心直觉

语言中有大量"简单" token（语法结构词、常见搭配、代码格式等），预测确定性很高，小模型和大模型预测几乎一致。让小模型猜，大模型验证一下，猜对了就省了一次前向传播 。

### 四步算法

1. **草稿生成**：草稿模型自回归生成 γ 个候选 token 
2. **并行验证**：目标模型一次性输入所有候选，一次前向传播得到所有位置的概率 
3. **接受-拒绝**：对每个候选按序验证，α_i = min(1, p_i/q_i)，采样 u_i ~ U(0,1)，若 u_i ≤ α_i 则接受 
4. **分布校正采样**：被拒绝时从校正分布 r(x) = max(0, p(x) - q(x)) 重新采样 

**关键保证**：投机解码生成的 token 序列服从目标分布 p(x)，输出分布完全不变 。

### 理论加速比

T_spec = γ·τ_q + τ_p（γ 次草稿推理 + 1 次目标模型验证）

| 场景 | α（接受率） | ρ（速度比） | γ_opt | 加速比 |
|------|-----------|-----------|-------|--------|
| 低延迟优先 | 0.7 | 0.1 | 4 | 1.8× |
| 平衡模式 | 0.8 | 0.1 | 6 | 2.3× |
| 高吞吐模式 | 0.9 | 0.05 | 10 | 3.5× |
| 理想情况 | 0.95 | 0.02 | 20 | 5.0× |

### 不同文本类型的接受率

| 文本类型 | α_avg | γ 推荐 |
|---------|-------|--------|
| 代码补全 | 0.80 | 6-8 |
| 技术文档 | 0.80 | 5-6 |
| 对话系统 | 0.75 | 4-5 |
| 创意写作 | 0.65 | 3-4 |
| 数学推理 | 0.60 | 2-3 |

### 与推理框架的关系

| 框架 | 实现方式 | 加速比 |
|------|---------|--------|
| 基础投机解码 | 独立小模型 + 大模型验证 | 1.7-3× |
| **MEDUSA** | One Model，增加多个解码头 | 2.2-3.6× |
| **EAGLE** | 特征级自回归头 | 2.7-3.5× |
| **EAGLE-2** | 动态草稿树 | 3.0-4.3× |
| **EAGLE-3** | 直接 token 预测 + 多层特征融合 | 峰值 6.5× |

演进核心思路：**草稿模型越接近目标模型的特征空间，接受率 α 越高，加速效果越好** 。
