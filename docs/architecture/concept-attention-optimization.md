## 摘要

针对标准缩放点积注意力 O(N²) 内存和反复 HBM 读写瓶颈的多种优化方案，包括精确计算类（Flash Attention、MQA/GQA）和近似计算类（稀疏注意力、线性注意力）。标准实现的 FLOPs 利用率仅 25-40% 。

## 详情

### 注意力复杂度瓶颈

标准注意力计算步骤 ：
1. `S = QK^T / √d` — 矩阵乘法，N×d × d×N → N×N
2. `P = softmax(S)` — 逐行归一化，N×N
3. `O = PV` — 矩阵乘法，N×N × N×d → N×d

总计算量：约 4N²d FLOPs，总内存：O(N²) 的中间矩阵。

**两大痛点** ：
| 痛点 | 说明 | 后果 |
|------|------|------|
| **内存墙** | 注意力矩阵大小 = O(N²) | 序列 8192 → 134MB（FP16），长序列爆显存  |
| **带宽墙** | 中间结果反复读写 HBM（S、P 矩阵写完再读） | FLOPs 利用率仅 25-40%  |

标准注意力实现 = 三次 HBM 往返 。

### Flash Attention：不写中间结果的聪明算法

**GPU 内存层次** ：
- SRAM（共享内存 / L1 Cache）：~192KB per SM（A100），带宽 ~19TB/s（极快） 
- HBM（高带宽内存 / 显存）：~40GB，带宽 ~1.5TB/s（慢 13 倍）

**核心思路**：分块计算（tiling），在 SRAM 内累积结果，不写中间矩阵，只跑一次书架 。

**增量 Softmax（在线算法）** ：
- 维护三个状态：m（当前最大值）、l（当前归一化因子）、O（当前输出累积）
- 每来一块新数据：更新最大值 → 缩放旧累积 → 加上新贡献 → 更新输出

**HBM 访问对比** ：
| 方法 | HBM 访问 | 复杂度 |
|------|---------|--------|
| 标准注意力 | 读写 S、P 矩阵 | O(N²) |
| Flash Attention | 只读 Q,K,V + 写 O | O(N²d²/M)，M 为 SRAM 大小 |

### Flash Attention v2 改进

| 改进项 | v1 | v2 | 效果 |
|--------|----|----|------|
| 工作分区 | sliced-K（拆分 K/V，warp 间需同步） | split Q（拆分 Q，warp 独立计算，消除通信） | 消除 warp 间共享内存读写  |
| 非 matmul | 多次 rescaling | 重写 online softmax，减少 rescaling | 更多计算落在 Tensor Core  |
| 序列并行 | 仅 batch/head 维度并行 | 新增序列长度维度并行 | 更好利用 GPU SM（小 batch 场景）  |
| 因果掩码 | 显式存储掩码 | 隐式判断 + 提前退出 | 减少计算  |

**FlashAttention-2 论文报告**：A100 上前向+反向合计最高 230 TFLOPs/s，端到端训练最高 225 TFLOPs/s（72% 模型 FLOPs 利用率） 。

### MQA / GQA：共享键值，减少冗余

标准 MHA 为每个头独立计算 Q、K、V，存在冗余 。

**KV Cache 瓶颈示例**：LLaMA-70B 假设使用 MHA（80 层，64 个 KV 头），batch=32, seq_len=2048 时 KV Cache ≈ 160 GB 。LLaMA-70B 实际使用 GQA，仅 8 个 KV 头，实际 KV Cache 约 20 GB 。

**MQA（Multi-Query Attention）**：所有查询头共享同一组 K 和 V 。
- KV Cache 减少 32 倍（以 32 头为例） 
- 解码速度提升 2-4x 
- 质量退化轻微：语言建模 PPL 约 1-3% 退化，翻译 BLEU < 1 下降 

**GQA（Grouped-Query Attention）**：把头分成 G 组，每组共享 KV 。

**LLaMA-70B 实验（64 个 Q 头）** ：
| 配置 | PPL 趋势 | KV Cache 比例 | 推理速度趋势 |
|------|---------|--------------|-------------|
| MHA（原始） | 基准 | 100% (64/64) | 基准 |
| GQA-8 | 几乎无损 | 12.5% (8/64) | 显著加速 |
| GQA-4 | 轻微退化 | 6.25% (4/64) | 明显加速 |
| MQA | 小幅退化 | 1.6% (1/64) | 最高加速 |

> **GQA-8 是甜点配置**：质量几乎无损，KV Cache 大幅降低，推理显著加速。LLaMA 2 70B 就用了 GQA-8 。

### 稀疏注意力：只算重要的连接

**核心观察**：注意力分布是长尾的——99% 的权重 < 0.01（几乎为零，可忽略） 。如果只算权重 > 0.01 的连接，计算量减少 99% 。

**窗口注意力（Window Attention）**：每个 token 只看左右 w 个邻居，计算量 O(Nw) 而非 O(N²) 。w=128 时，比全注意力快 ~64 倍（N=8192） 。

**BigBird 组合模式** ：
1. 窗口注意力：看附近邻居（局部信息）
2. 随机注意力：随机选 r 个 token 连接（打通信息孤岛）
3. 全局注意力：特殊 token（如 [CLS]）连接所有人（全局汇总）

BigBird 的稀疏注意力图直径为 O(log N)，有利于信息在层间快速混合；含全局 token 的稀疏注意力是序列函数的通用逼近器（Universal Approximator） 。

> **范畴论视角**：从完全稠密注意力（完全图大范畴）中只保留部分位置间的箭头，本质是在大范畴中选择一个子范畴。不同注意力变体的差异可归结为"谁多保留了什么箭头、谁多砍掉了什么箭头" 。参见 [范畴论与 LLM 系统思维](/base-models/concept-category-theory-llm)。

**复杂度对比** ：
| 模式 | 连接数/token | 总复杂度 | 稀疏度 |
|------|-------------|---------|--------|
| 全注意力 | N | O(N²) | 0% |
| 窗口 (w=128) | 2w+1 | O(Nw) | ~98% |
| BigBird (w=3,r=2,g=2) | ~13 | O(N) | ~97.5% |

### 线性注意力：改变计算顺序

**核心思想**：改变计算顺序 。
- 标准顺序：`(Q × K^T) × V` → 中间矩阵 N×N
- 线性顺序：`Q × (K^T × V)` → 中间矩阵 d×d

**RNN 形式的线性注意力** ：
```
S_i = S_{i-1} + φ(K_i) × V_i^T    ← 状态累积
O_i = φ(Q_i) × S_i / 归一化        ← 用状态算输出
```

关键：S_i 是固定大小 d×d 的矩阵！不需要存储历史，一个状态就够了 → O(1) 内存，真正的流式推理 。

**常见 φ 选择** ：
- Linear Transformer: φ(x) = elu(x) + 1
- Performer: 随机特征映射

**优缺点** ：
| 维度 | 优点 | 缺点 |
|------|------|------|
| 计算复杂度 | O(Nd²)，N 大时优势明显 | N 小时不如标准 |
| 内存 | 流式推理 O(1) | 需要存储状态 S |
| 质量 | 局部依赖任务好 | 全局推理任务退化 |

**混合策略（推荐）**：前几层用标准注意力（捕获精确模式），后几层用线性注意力（处理长距离依赖），质量轻微退化，但长序列显著加速 。

### 四种优化方案对比

| 方案 | 核心思路 | 计算精度 | 加速比 | KV Cache | 典型应用 |
|------|---------|---------|--------|---------|---------|
| Flash Attention | 分块 + 增量 softmax | **精确** | 1.5-3× | 不变 | 训练/推理通用  |
| MQA | 所有头共享 KV | **精确** | 1.5-4× | **1/H** | 大模型推理  |
| GQA | 分组共享 KV | **精确** | 1.5-2× | **G/H** | LLaMA 2/3  |
| 稀疏注意力 | 只算重要连接 | 近似 | 2-5× | 不变 | 长文档理解  |
| 线性注意力 | 改变计算顺序 | 近似 | 3-10× | **O(1)** | 流式推理  |

### 组合使用

**最优边缘部署配置** ：

预填充阶段：Flash Attention（精确计算，加速 1.5-2×）+ GQA-8（KV Cache 降 1/8）

解码阶段：GQA-8（推理显著加速）

超长序列（>4K）：
- 前 4 层：标准注意力 + Flash（精确局部模式）
- 中间 4 层：GQA-4（平衡效率）
- 后 4 层：窗口(256) + 线性混合（省计算）
