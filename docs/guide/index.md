# 如何阅读本知识库

本知识库是一套结构化的 AI 工程知识体系，覆盖从基础原理到工程落地、从模型架构到行业实战的完整知识链路。以下指南帮助你根据自身需求找到最高效的学习路径。

## 知识库结构总览

本知识库按主题分为 **9 大模块**，每个模块下包含若干 concept（概念页）、topic（主题综述）和 model/product（实体档案）：

| 模块 | 内容主题 | 适合人群 |
|------|----------|----------|
| [基础模型与训练](../base-models/) | Transformer、Embedding、推理模型、LLM 训练范式 | 所有人（建议从这里开始） |
| [模型架构](../architecture/) | 注意力优化、KV Cache、MoE、解码加速 | 算法工程师、架构师 |
| [训练与优化](../training-optimization/) | 微调、量化、剪枝、蒸馏、RLHF | 模型工程师 |
| [推理与部署](../inference-deploy/) | GPU 选型、高并发、推理框架、边缘部署 | 后端/DevOps 工程师 |
| [Agent 智能体](../agent/) | 设计模式、MCP/A2A 协议、评估体系 | Agent 开发者 |
| [RAG 检索增强](../rag/) | 检索策略、分块、Query 改写、Agentic RAG | 应用开发者 |
| [多模态与视觉](../multimodal/) | 图像/视频生成、视觉识别、数字人 | 多模态开发者 |
| [行业应用案例](../industry-cases/) | 财务、HR、直播、教育场景实战 | 产品经理、业务负责人 |
| [工具与生态](../tools-ecosystem/) | GPU 硬件、CUDA、芯片竞争、经典论文 | 所有人（横向视野） |

## 推荐阅读路径

### 路径一：快速入门（1-2 天）

适合想快速了解 LLM 全貌的初学者：

1. [基础模型与训练](../base-models/) → Transformer 架构、LLM 发展简史、训练范式
2. [RAG 检索增强](../rag/) → RAG 基础原理
3. [Agent 智能体](../agent/) → Agent 开发全览、五大架构模式

### 路径二：AI 工程师（1-2 周）

适合需要进行模型选型、训练优化和部署的工程师：

1. [基础模型与训练](../base-models/) → 完整阅读
2. [架构](../architecture/) → 注意力优化、KV Cache、显存计算
3. [训练与优化](../training-optimization/) → 微调、量化、蒸馏
4. [推理与部署](../inference-deploy/) → GPU 选型、框架选择、高并发

### 路径三：Agent 开发者（1 周）

适合构建 AI Agent 应用的开发者：

1. [Agent 智能体](../agent/) → MCP 协议、Function Calling、五大架构
2. [RAG 检索增强](../rag/) → Agentic RAG、高级检索技术
3. [推理与部署](../inference-deploy/) → AI 框架选型

### 路径四：业务落地（3-5 天）

适合产品经理和业务负责人：

1. [基础模型与训练](../base-models/) → Embedding 选型、LLM 发展简史
2. [行业应用案例](../industry-cases/) → 完整阅读
3. [多模态与视觉](../multimodal/) → 数字人、视频生成

## 页面类型说明

本知识库中有三种页面类型，通过文件名前缀区分：

- **`concept-`**：概念解析页，深入讲解单一技术概念，是可读性最强的内容
- **`topic-`**：主题综述页，汇总一个领域的完整方法论和知识链路
- **`model-` / `product-`**：实体档案页，记录具体模型、GPU 或产品的技术参数

## 使用建议

- **按需跳读**：每个页面独立成篇，可以从任何位置切入，不需要按顺序阅读
- **Topic 先行**：建议先读 `topic-` 综述页了解全局，再按兴趣深入 `concept-` 细节
- **交叉引用**：页面间通过 `[[]]` 互为引用，发现关联时不妨跳转探索
- **关注更新**：每个页面的元数据区标注了创建和最后更新时间，优先关注近期更新内容
