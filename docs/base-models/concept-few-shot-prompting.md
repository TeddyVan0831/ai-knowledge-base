# Few-Shot Prompting

> **类型**: concept
> **创建时间**: 2026-05-27
> **最后更新**: 2026-05-27
> **来源**: 

## 摘要

Few-Shot Prompting 在系统提示词中加入优秀回答样例（如 3 条样例 = 3-shot），显著提升模型回答质量，是理解 RAG 检索增强必要性的前置概念。

## 详情

### 核心原理

在提示词中塞样例能较好地控制大模型的回答，模型表现相对稳定。但核心问题是提示词字数有限，用户可能问的问题太多，无法手工预设所有样例——引出动态检索的需求 。

### 索尼门店案例

**Zero-Shot（无样例）**：仅设定角色和目标（"你是索尼门店店员，目标是卖东西，言简意赅，不超过300字，口语化"），模型直接回答。结果：回答"不算错，但绝对不算优秀"——罗列通用信息 。

**Few-Shot（3 条样例）**：加入优秀回答样例后，模型回答质量显著提升 ：

> "Switch 侧重便携性，随时随地就能玩，这是它最大的好处。但家里肯定有手机和 Pad，同样是便携设备，手机游戏丰富多了，专门的 Switch 就显得可有可无。PS5 定位客厅硬核体验，沉浸感手机和 Switch 都给不了。"

### 问题分类的价值

真实场景中最优秀的 top 级店员会先对问题**分类**——如"竞品比较"下的"价格比较"子分支，然后套用最佳实践 。Few-Shot 样例本质上就是在教模型做这种分类和套用。
