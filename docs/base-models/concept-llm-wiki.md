## 摘要

LLM Wiki 是由 Karpathy 于 2026 年 4 月提出的结构化知识系统架构——用"页面化"替代传统 RAG 的切片方式，通过双向链接、目录、摘要和混合搜索构建网络化的知识系统，将搜索从"单点匹配"升级为"导航探索"。

## 详情

### 解决的问题

传统向量 RAG 的痛点：文档切片导致碎片化——切小了语义不完整，切大了检索精度下降 。

**核心思路**：从"单点碎片匹配"到"网络系统导航" 。

### 七个核心模块

| 模块 | 说明 |
|------|------|
| **页面化（Pageification）** | 告别切片，把每个主题做成一整页，包含 title、content、summary  |
| **双向链接（Bidirectional Links）** | 从每个页面提取 100 个实体/概念关键词，逐一检测是否已有对应页面，建立正向和反向链接  |
| **结构化目录（Directory）** | 基于所有页面的标题 + summary，由 LLM 自动生成目录，在检索中用于路径导航  |
| **实体关系** | 可选模块，经典知识图谱三元组，用于多跳推理问题  |
| **自动摘要** | 多层级自适应摘要（L1 标题→L2 短摘要→L3 中摘要→L4 原文），根据文档字数动态调整  |
| **混合搜索（Hybrid Search）** | 向量检索 + 关键词检索 + 实体双向链接检索 + 目录导航 + 时间过滤 + 权限控制  |
| **Agentic Search** | ReAct Agent 自主决定检索策略（先翻目录？先搜关键词？先查向量？），循环执行直到信息足够  |

### 三层架构

```
Schema 层（规则层）：文字规则、代码过滤、清单、摘要格式规范
  ↓
Wiki 层（编译后）：页面文件、双向链接、目录、向量索引、知识图谱
  ↓
Raw 层（原始数据层）：PDF、网页、抓取内容，含冗余信息
```

### 与知识图谱的区别

| 维度 | 经典知识图谱 | LLM Wiki 双向链接 |
|------|------------|------------------|
| 连接对象 | 实体 ↔ 实体 | 关键词 → 页面 |
| 关系属性 | 有明确属性标签（就职/负责/服务） | 无属性标签（就是跳转） |
| 结构化程度 | 高（三元组） | 低（超链接式） |
| 学术称呼 | 结构化知识图谱 | **非结构化知识图谱** |

### 与 RAG 的关系

LLM Wiki 不能替代 RAG——它本质上仍是广义 RAG，只不过检索方式从单一向量变成了混合搜索 。
