# 大语言模型训练范式

> **类型**: concept
> **创建时间**: 2026-05-28
> **最后更新**: 2026-05-28
> **来源**: 

## 摘要

LLM 训练采用分阶段范式——同一模型在不同阶段使用不同数据和目标逐步提升能力：预训练（学语言规律）→ SFT（学做任务）→ 对齐（RM + RL 或 DPO）。

## 详情

### 底层认知

大语言模型本质上只做一件事：**续写** 。预训练模型像"接话茬"，SFT 阶段才开始教模型做任务。

### 当前主流训练范式（全球 80%+ 模型采用） 

```
预训练 (Pretrain) → SFT → Reward Model → 强化学习 (PPO)
                                               ↕ 循环迭代
```

### 四个阶段对比

| 阶段 | 训练目标 | 数据形式 | 数据量 | 模型能力 |
|------|---------|---------|--------|---------|
| **预训练** | 段落联合概率最大化 | 纯文本段落 | 几十亿条（Llama 3: ~39 亿 ） | 懂语言规律，会"接话茬" |
| **SFT** | 段落联合概率最大化（仅回答部分） | 一问一答 | 几万到百万条（差 5000-10000 倍） | 会做任务，指令遵循 |
| **Reward Model** | 符合人类排序偏好 | 偏好排序数据 | 取决于标注规模 | 能给回答打分 |
| **PPO** | RM 打分综合得分最大化 | 提示词 + RM 反馈 | 数十万到百万条 | 对齐人类期望 |

### 关键概念

- **预训练是自监督**：答案在原始数据里（下一个字就是正确答案） 
- **SFT 与预训练目标相同**：都是段落联合概率最大化，区别仅在于问题部分不参与计算 
- **对齐 = 与人类期望对齐**：又严谨又全面又有深度又有洞察 

### 两种路径

| 维度 | GPT 路径 | LLaMA 路径 |
|------|----------|-----------|
| 对齐核心方法 | RM + PPO 强化学习 | Rejection Sampling + DPO |
| RM 的用途 | 在 RL 中给主模型打分 | 做数据筛选 |
| 迭代方式 | RM ↔ RL 互相训练循环 | 全流程大循环（~6 轮） |
