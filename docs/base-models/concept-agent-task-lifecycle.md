# Agent 任务生命周期

> **类型**: concept
> **创建时间**: 2026-07-05
> **最后更新**: 2026-07-05
> **来源**: 
> **领域**: ai

## 摘要

Agent 任务不是一次模型调用，而是一段**有状态的生命周期**。它由 8 个阶段组成（任务接入 → 重新定位 → 规划与路由 → 行动 → 观察 → 验证 → 持久化 → 结束/升级），每阶段都可能因验证失败、上下文压力、权限拒绝或任务变化而回到前面任一阶段。理解生命周期是从"任务问题"反推"系统责任"的核心方法。

## 详情

### 一、Agent 任务与单次调用的 6 大差异

| # | 差异 | 系统新增责任 |
|---|------|------------|
| 1 | **输出变成行动** | token 序列对应文件修改、转账、工单、外部消息 |
| 2 | **一次调用变成持续循环** | 错误会累积；需预算、停止、重试、升级 |
| 3 | **外部环境成为任务状态一部分** | 文件、进程、浏览器、数据库持续变化 |
| 4 | **上下文窗口 ≠ 任务记忆** | 会话会压缩、中断、重启、并发 |
| 5 | **最终回复 ≠ 交付结果** | 完成需要独立证据（测试、断言）|
| 6 | **调用接口变成长期服务** | 消息、流式、取消、并发、身份、审计 |

### 二、生命周期 8 阶段

```
任务接入 → 重新定位 → 规划与路由 → 行动 → 观察 → 验证 → 持久化 → 结束/升级
   ↑          ↑          ↑          ↑       ↑       ↑        ↑          ↓
   └──────────┴──────────┴──────────┴───────┴───────┴────────┴──────────┘
              (任意阶段可因验证失败/上下文压力/权限拒绝/任务变化而回退)
```

| 阶段 | 主要内容 |
|------|---------|
| 任务接入 | 目标、范围、证据 |
| 重新定位 | 工作空间、进度、基线 |
| 规划与路由 | 分解、选择模型/技能 |
| 行动 | 工具、消息、外部系统 |
| 观察 | 结果、事件、错误 |
| 验证 | 测试、策略、评审 |
| 持久化 | 状态、记忆、交接 |
| 结束/升级 | 交付、取消、转人工 |

### 三、阶段问题与系统保障对照

#### 任务开始前

| 常见问题 | 系统保障 |
|---------|---------|
| 只写"做什么"，没写"什么算完成" | **任务契约**：意图、范围、验收证据、预算、升级条件 |
| Workspace 半成品，Agent 按空项目理解 | **Orient**：读取目录、版本、进度、基线测试、未完成列表 |
| 模型不知道权限边界 | **运行时身份绑定** + 最小权限 + 网络/数据范围 |
| 一开始加载全部文档和工具 | **渐进式披露**：按任务加载 Skills、知识和工具定义 |

#### 行动过程中

| 常见问题 | 系统保障 |
|---------|---------|
| 选错工具或参数 | 清晰命名、Schema、类型校验、可行动错误信息 |
| 超时后重复执行写操作 | 幂等键、状态查询、事务、重试策略、回滚 |
| 循环持续但没有进展 | step/time/cost budget、no-progress 检测、重新规划 |
| 试图执行高风险动作 | Policy gate、审批、只读/可逆/不可逆分级 |
| 工具输出淹没上下文 | 过滤、分页、截断、摘要与 Context manifest |

#### 长任务与服务化阶段

| 常见问题 | 系统保障 |
|---------|---------|
| Context reset 后重复劳动 | Progress、Plan、Decision log、Checkpoint、Handoff |
| 进程崩溃后从头开始 | Workspace snapshot、可重放事件、恢复状态机 |
| 外部系统持续发来消息 | 会话身份、消息收发、队列、并发控制、取消 |
| 流式输出和后台任务无法关联 | Event stream、Run ID、状态事件、消费端协议 |
| 模型宣布完成但结果不完整 | Tests、状态断言、独立 Reviewer、人工升级 |
| 失败无法复现和改进 | Trace、工具调用、环境快照、Harness 版本、Eval 回归 |

### 四、生命周期驱动的 6 大系统模块

| 模块 | 典型内容 | 主要解决的问题 |
|------|---------|------------|
| **模型交互与输出控制** | Structured Output、Function Calling、模型路由、上下文入口 | 把开放式生成转成系统可消费的决策与数据 |
| **编排与行动 Loop** | 状态机、路由、计划、重试、停止、子任务、多 Agent | 让任务按观察结果持续推进，保持边界和预算 |
| **工具、Skills 与 Workspace** | MCP/API、Shell、文件、浏览器、Skill、Sandbox、快照 | 让模型获得行动力和过程知识，限制副作用 |
| **会话、状态与记忆** | Session、Progress、Checkpoint、短期/长期记忆、Handoff | 多轮/多会话/长任务中维持连续性 |
| **服务化与消息边界** | API/Worker、外部消息、Streaming、Cancel、Concurrency | 把 Agent 作为长期运行的信息系统能力对外提供 |
| **可观测、验证与 Eval** | Event logs、Trace、解析与可视化、Tests、Policy、Eval、Regression | 还原过程、独立验收、归因失败、持续改进 |

> 身份、权限、成本、安全、审计、版本治理 **贯穿所有模块**。
