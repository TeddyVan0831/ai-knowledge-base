# 推理模型与 DeepSeek R1：从 R1-Zero 到多阶段训练

> **类型**: concept
> **创建时间**: 2026-06-09
> **最后更新**: 2026-06-09
> **来源**: 

## 摘要
推理模型是在通用大模型基础上通过强化学习等方式训练出的、擅长多步复杂推理的模型。DeepSeek R1 的底座是 DeepSeek V3（671B MoE），本质是**强化学习后的 V3**。R1 的训练经历了**R1-Zero（纯 RL 突破）→ 冷启动 SFT → GRPO → 蒸馏 SFT → 最终 GRPO** 的多阶段流程，最终在 AIME 2024 上达到 79.8%，与 OpenAI O1 性能相当 。

## 核心概念

### 推理模型 vs 通用模型

| 维度 | 通用模型（如 DeepSeek V3） | 推理模型（如 DeepSeek R1） |
|------|---------------------------|---------------------------|
| **本质** | 底座基座模型 | 在通用模型基础上训练而来 |
| **擅长场景** | 通识问答、翻译、摘要、分类 | 数学推理、编程、多步逻辑问题 |
| **输出特点** | 直接给出答案 | 先输出思维链（`<think>` 标签包裹），再给答案 |
| **速度** | 较快 | 较慢（需输出思维链） |
| **激活参数** | 37B / 671B（MoE 架构） | 同 V3（37B / 671B） |  |

**关键认知**：
- R1-Zero 和 R1 的底座都是 DeepSeek V3，R1 本质是强化学习后的 V3
- 推理模型**不适合所有场景**——简单任务不仅慢，还可能因过度思考而偏离指令 

### R1 爆发时间线与破圈原因

| 时间 | 事件 |
|------|------|
| 2024-11-20 | R1-Light 预览版上线 |
| 2024-12-26 | V3 底座模型发布（性能为 V2 的 3 倍） |
| 2025-01-15 | DeepSeek App 上线 |
| 2025-01-20 | **R1 模型正式发布** |

App 发布 20 天内日活突破 2000 万，2 月 5 日 DAU 突破 4000 万（外部端 1600 万） 

**破圈的三个原因**：
1. C 端体验突破：中文文创能力强，思维链让用户感到"模型在思考"
2. 开源同类首个：首次将类 O1 推理能力以开源方式呈现给大众
3. 国际舆论催化 + 行业人物转向（如朱啸虎从"大模型没戏"到"相信 AGI 到来"） 

### R1-Zero：纯强化学习的突破

- 直接以 671B DeepSeek V3 底座模型为起点，**不使用任何人工标注的 SFT 数据**，纯粹通过强化学习训练
- 这是业界首个验证了"仅用 RL 即可让大模型获得强推理能力"并开源的工作
- **奖励模型**：纯规则化（不依赖神经网络）
  - 准确度奖励：答案是否正确（编译器验证/数学标准答案比对）
  - 格式奖励：是否将思考过程` 标签内，答案放在 `<answer>...</answer>` 标签内
- **关键实现**：训练模板要求模型将最终答案包裹在 `\boxed{}` 标签中，规则奖励函数通过解析自动提取答案进行比对 

**Aha Moment（顿悟时刻）**：
- 随着强化学习训练步数增加（几千步后即表现出色），模型出现了类似人类的自我反思行为
- 思维链长度可达 8,000-10,000 Token
- 在 AIME 2024 上的表现从初始的 **15.6%** 飙升至 **71.0%**（接近当时 OpenAI O1 的 74.4%）
- 这些行为是模型在 RL 过程中**自发涌现**的，不是人工编写的 

**R1-Zero 的两大问题**（因此未直接发布）：
1. 可读性差：思考过程对人类难以阅读
2. 语言混杂：思维链中英文夹杂 

### R1：多阶段训练流程

```
V3 底座
  ├─[冷启动 SFT]── R1-Zero 生成的长思维链数据 + 人工 refine
  ├─[第1轮 GRPO]── 强化学习增强推理（新增语言一致性奖励）
  ├─[第2轮 SFT]── R1 蒸馏数据（推理 600K）+ V3 生成数据（通用 200K）
  └─[第2轮 GRPO]── 最终强化学习 → DeepSeek R1
```

### GRPO 算法核心

GRPO（Group Relative Policy Optimization）与 PPO 的核心改进：

| PPO | GRPO |
|-----|------|
| 对一个 prompt 只采样 1 个回答 | 对一个 prompt 采样**一组**（Group）回答 |
| Reward Model 是神经网络 | Reward 是**规则函数**（对/错/格式） |
| 需要 Value Model（占一半显存） | **完全砍掉 Value Model** |
| 4 个模型 | **3 个模型** |  |

- **群组打分机制**：对一个 prompt 采样 G 个回答，计算均值、标准化，得到每个回答的"相对优势"
- **优势**：砍掉 Value Model，显存减半；纯规则奖励避免 Reward Hacking
- **局限**：只能用于有标准答案的任务（数学、编程） 

### 知识蒸馏

R1 论文中的"蒸馏"是**数据蒸馏**（非传统算法层面 loss 对齐）：
1. 用 R1 生成高质量 SFT 数据（含思维链）
2. 直接用这些数据 SFT 训练 LLaMA 和 Qwen 的稠密模型（1.5B/7B/8B/14B/32B/70B，全部开源）
3. Qwen-32B 蒸馏版 AIME 达 72.6%——逼近 R1 本身的 79.8%
