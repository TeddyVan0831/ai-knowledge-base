# ORM：结果监督奖励模型

> **类型**: concept
> **创建时间**: 2026-06-12
> **最后更新**: 2026-06-12
> **来源**: 

## 摘要

ORM（Outcome-supervised Reward Model，结果监督奖励模型）是 WebRL 强化学习框架的核心组件。本质是一个经过微调的语言模型，能够评估智能体在执行任务过程中的操作路径，并提供二进制奖励信号（0 或 1）。在 WebArena-Lite 测试集上准确率达 **80.8%**，超过 GPT-4（71.9%）和 GPT-4V（71.2%）。

## 详情

### 定义与作用

ORM 训练一个 LLM，使其能够评估智能体执行任务过程中的操作路径，提供二进制奖励信号（0 或 1）。这解决了网页操作场景中的**反馈信号稀疏**问题——不像游戏有明确的"赢/输"判定，网页操作很难自动判断是否成功。

### ORM 的输入

三个要素：

| 输入 | 说明 |
|------|------|
| 任务指令 | 用户最初的任务描述（如"在美团上点一杯大杯热美式"） |
| 历史行动 | 智能体在中间步骤执行的所有操作记录 |
| 最终状态 | 最后一个网页的 HTML 内容（简化后） |



### ORM 的输出

- 输出两个概率值：成功概率（Yes）和失败概率（No）
- 当 Yes 概率 > 50% 时，认为任务成功，奖励 = 1
- 当 Yes 概率 ≤ 50% 时，认为任务失败，奖励 = 0



### 训练方式

- 基于现有语言模型进行微调（后训练）
- 将其从"生成文字"改为"二分类判别"
- 可判断特定任务的完成状态（如知乎文章发布后是否到达发布成功页面）

**训练数据**（论文实际使用）：
- 收集了 **12,200 个** 交互轨迹样本
- 来源：WebArena-Lite 增强数据集 + WebArena 补充样本 + 基线方法和 WebRL 的交互轨迹
- 训练超参数：学习率 5e-6，cosine 调度器，batch size 128，训练 4 轮



### 评估准确率

| 方法 | 测试集准确率 | Rollout 准确率 |
|------|------------|---------------|
| **WebRL ORM (8B)** | **80.8%** | **79.4%** |
| GPT-4 (text-only) | 71.9% | 71.2% |
| GPT-4V | 71.2% | 70.5% |



### 与 RLHF 的关系

ORM 与 RLHF 形成互补：

- **ORM**：自动化、大规模的二进制奖励（0/1），适合规模化训练
- **RLHF**：人工精细化的质量评估，提供更细致的人类偏好信息
- AutoGLM 产品界面中的"点赞/踩"反馈可被收集用于 RLHF 

> 授课人评价：ORM 的训练"挺简单的"，因为只要判断最终页面是否符合任务完成的结果即可。
