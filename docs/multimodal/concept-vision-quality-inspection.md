## 摘要

本文涵盖工业视觉检测的完整技术栈：从传统图像识别方法、CNN 深度学习、YOLO 目标检测算法，到 VLM（视觉语言模型）多模态方案在工业质检中的应用，以及 YOLO 与 VLM 的互补关系和基于 Bad Case 的持续迭代闭环。

## 详情

### 一、视觉检测技术演进

**人脑识别模型**：分级的、多层网络模型，从 Pixels（像素）→ Edges（边缘）→ Object Parts（物体部件）→ Object Models（物体模型），减少数据量，保留有用信息 。

**传统图像识别方法**：依靠特征工程，制定规则判断目标是否符合标准，代表软件包括 Halcon、Visionpro、OpenCV 。流程为：数据获取 → 预处理 → 特征提取 → 决策分类 。局限性在于检测物体变动时规则和算法需重新设计，不同批次产品变化难以通用 。

**深度学习视觉检测**：采用深度神经网络（CNN），自动学习特征，无需人工设计 。但数据量要求大，工业数据收集成本高，解决方案包括数据增强、无监督学习、自动数据标注 。

### 二、CNN 发展里程碑

ImageNet 包含超过 15 million 图像，约 22,000 类 。ILSVRC 竞赛 2010-2017 年举办 ，历届 CNN 里程碑如下 ：

| 年份 | 模型 | Top-5 Error |
|------|------|-------------|
| 2012 | AlexNet | 15.3%  |
| 2014 | VGGNet | 7.3%  |
| 2014 | GoogLeNet | 6.7%  |
| 2015 | ResNet | 3.57%  |
| 2017 | SENet | ~2.3%  |



### 三、YOLO 目标检测算法

**目标检测算法分类**：
- 基于 Region Proposal 的 R-CNN 系列（如 R-CNN、Fast R-CNN、Faster R-CNN）属于 **two-stage**，需要先使用启发式方法或 CNN 网络产生 Region Proposal，再在 Region Proposal 上做分类与回归 
- YOLO、SSD 这类 **one-stage** 算法，只使用一个 CNN 网络直接预测不同目标的类别与位置 
- **Two-stage 方法准确度高一些，但速度慢，one-stage 算法是速度快，但是准确性低一些** 

**YOLO-v1 核心思想**：将检测变为回归问题，输入图像只经过一个 neural network，直接得到 bounding boxes 以及每个 bounding box 所属类别的概率 。步骤包括：将图片分割为 S×S 个 Grid（如 7×7），每个 Grid 预测 B 个 Bounding Box，每个 Box 预测 5 个值 (x, y, w, h, confidence)，每个 Grid 预测 C 个类别概率 。输出张量维度为 S × S × (B × 5 + C) 。

**置信度计算**：`confidence = Pr(Object) × IOU_pred^truth` 

**非极大值抑制（NMS）**：选取 scores 最大的 Box，计算与其余 Box 的 IOU，若 IOU > threshold（如 0.5），舍弃该 Box 。

**YOLO-v1 优点**：速度快（回归方法），mAP 高（63.4% vs Fast YOLO 52.7%），基于整张图片信息预测 。
**缺点**：每个 Grid 最多预测一个物体（小物体密集场景不适用），输出层为全连接层，只支持训练时相同的输入分辨率 。

**YOLO 速度**：45-155 FPS 。

### 四、工业缺陷检测方案

**制定方案考虑因素**：数据量（能获取多少数据，尤其是缺陷数据）、人工判断（将人工判异方法量化）、公开图片（数据量不足时考虑类似的公开图片进行迁移学习）。

**硬件选择**：推荐使用黑白相机（算法基于灰度值处理），相同分辨率下黑白相机精度更高 。工业相机传感器更大（邮票大小），手机传感器小（指甲盖大小）。

**光源选择**：环形光（圆形或方形物体）、条形光（矩形物体，照射角度要求高）、背光（轮廓尺寸测量）、线光（配合线阵相机使用）。

**钢铁缺陷检测**：识别 6 类缺陷（crazing 开裂、inclusion 夹杂、pitted_surface 点蚀、scratches 划痕、patches 斑块、rolled-in_scale 氧化铁皮压入）。数据集训练集 1400 张（200×200 像素），测试集 400 张 。

**评估指标**：Precision = TP / (TP + FP)，Recall = TP / (TP + FN)，AP 为 PR 曲线下面积，mAP 为各类别 AP 的平均值，F1 = 2 × (Precision × Recall) / (Precision + Recall) 。TP 判定标准为 IOU > 0.5 。

### 五、VLM 多模态方案

**VLM 的独特价值**：
- **零样本识别**：不需要训练数据，改 Prompt 就能识别新缺陷 
- **自然语言解释**：可以描述缺陷特征，给出判断依据 
- **快速验证**：新产品上线初期先用 VLM 顶着 

**VLM 检测流程**：输入图片 → Base64 编码 → 构造 Prompt → 调用 VLM API → 解析 bbox_2d 响应 → 坐标转换 → 画框标注 。示例中使用 qwen3-vl-plus，推理耗时 18 秒，检测到 8 个缺陷 。

**Qwen-VL 坐标格式演进**：

| 版本 | 定位格式 | 坐标范围 | 说明 |
|------|----------|----------|------|
| Qwen-VL（初代） | `<ref>文字</ref><box>(x1,y1),(x2,y2)</box>` | 0-1000 | 离散 token，需正则提取 |
| Qwen2.5-VL | `{"bbox_2d": [x1,y1,x2,y2]}` | 相对于 resized 输入的绝对像素 | 需根据 image_grid_thw 计算 resized 尺寸后映射回原图 |
| Qwen3-VL | `{"bbox_2d": [x1,y1,x2,y2]}` | 0-1000 | 归一化值，需转换 |



**坐标转换关键差异**：Qwen3-VL 坐标归一化到 0-1000，必须 × 原图尺寸 / 1000 。Qwen2.5-VL 坐标不是原图绝对像素，而是相对于模型内部 resized 输入的尺寸，需根据 `image_grid_thw` 计算模型实际处理的图片尺寸，再按比例映射回原图 。

### 六、YOLO vs VLM 对比与互补

| 维度 | YOLO (YOLOv12n) | VLM (Qwen-VL) |
|------|-----------------|---------------|
| 定位精度 | 像素级 bbox | 有 bbox_2d（归一化坐标） |
| 推理速度 | ~10ms/张（GPU） | ~2-5s/张（API） |
| 分类准确率 | 高（训练过的类别） | 中（零样本，可能误判） |
| 新缺陷适应 | 需新数据 + 重训练（天/周） | 改 Prompt 即可（分钟） |
| 部署成本 | 本地 GPU/CPU，无额外费用 | API 有调用费用 |
| 可解释性 | 只输出框和分数 | 可解释"为什么" |



**工业质检协作模式**：产线实时检测 → YOLO（速度快、精度高、可离线部署），可疑/低置信度结果 → 辅助分析复审 → VLM（零样本、自然语言解释、快速适应新场景）。YOLO 与 VLM 不是替代关系，而是互补关系 。

### 七、持续改进闭环

**模型迭代频率**：
- 新产线上线初期：每 1-2 周，Bad Case > 50 张 
- 产线稳定运行：每 1-3 个月，Bad Case > 200 张或漏检率上升 
- 发现新缺陷类型：立即，出现训练集中没有的缺陷 

**竞赛 vs 工业应用差异**：
| 维度 | 竞赛 | 工业 |
|------|------|------|
| 目标 | mAP 分数最高 | 漏检率最低 + 满足产线节拍 |
| 置信度策略 | 调到 mAP 最优 | 偏低（宁可多检不可漏检） |
| 速度要求 | 不关注 | 硬约束（如 < 100ms/张） |
| 模型选择 | 可用大模型 + 集成 | 小模型优先（nano/small） |
