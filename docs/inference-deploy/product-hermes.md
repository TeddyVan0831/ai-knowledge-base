# Hermes

> **类型**: entity
> **创建时间**: 2026-05-26
> **最后更新**: 2026-05-26
> **来源**: 

## 摘要
Hermes 是在 OpenClaw 基础上升级的"管家级"智能体系统，通过四级分层记忆、心跳增强、Skill 增强、AI coding 增强、任务并行和任务失败隔离等 7 大核心升级，实现从"能做事"到"管家级"的进化。

## 详情

### 七大核心升级
1. **四级分层记忆** 
2. **心跳增强（6 大差异）** 
3. **Skill 增强（5 大对比）** 
4. **AI coding 增强（4 大对比）** 
5. **任务并行** 
6. **任务失败隔离** 
7. **核心提示词优化** 

### 四级分层记忆架构
Hermes 每学到一段东西，都会自动切成 **4 个粒度**：

| Tier | 名称 | 内容来源 | 说明 |
|------|------|----------|------|
| **Tier 0** | 标题/核心关键词 | 从 Tier 1 摘要里自动提取 | 仅标题/核心关键词，最精简  |
| **Tier 1** | 摘要 | 对原始全文做压缩总结 | 原文 **2000 字 → 压缩成 100~200 字** 要点  |
| **Tier 2** | 完整内容 | 当天完整对话日志 | 最原始的内容本体  |
| **Tier 3** | 深层参考 | 自动关联的相关历史记忆、技能文档、代码片段、API 文档等 | 扩展资料，提供深度上下文  |

**存储方案**：所有聊天记录、会话历史、结构化状态 → 全进 **SQLite**，优先 **LanceDB**，也可降级到 `memory/YYYY-MM-DD.md` + `MEMORY.md` 

### OpenClaw vs Hermes 记忆对比
| 维度 | OpenClaw（两级） | Hermes（四级） |
|------|-----------------|---------------|
| **结构** | 每日记录 + 长期记忆 | 标题 → 摘要 → 全文 → 深层参考  |
| **精度** | 粗粒度，手动提炼 | 细粒度，自动分层压缩  |
| **检索** | 关键词搜索、向量搜索 | 四级向量分别嵌入，按查询深度匹配  |
| **存储** | Markdown 文件为主 | JSON 结构化存储 + 向量数据库  |

### 心跳增强（6 大差异）
| 差异维度 | OpenClaw | Hermes |
|----------|----------|--------|
| **状态持久化** | 状态基本存在内存，重启后丢失 | 用 `heartbeat-state.json` 持久记录  |
| **智能频率控制** | 固定间隔，简单判断时间 | **动态调整心跳频率**：30 分钟内刚查过 → 跳过；主人活跃/忙碌 → 降低频率  |
| **主动工作深度** | 简单任务 | **自动做维护类工作**：整理记忆、优化技能、反思错误、检查 Git、更新文档 → **管家级**  |
| **心跳与 Cron 分工** | 心跳和定时任务混在一起 | **清晰分离**：Heartbeat（批量、轻量、允许漂移、上下文感知）vs Cron（精确时间、独立任务） |
| **上下文感知** | 只看时间、安静时段 | **能感知**：主人是否在忙、最近对话强度、是否适合主动推送  |
| **记忆联动** | 心跳和记忆系统弱关联 | **心跳 = 记忆维护的核心时机**：清理过期、摘要每日记录、更新长期记忆  |

### Skill 增强（5 大对比）
| 对比 | OpenClaw | Hermes |
|------|----------|--------|
| **自动提炼** | 需要你主动触发（半自动 + 人工引导） | **完全静默、全自动**：做完复杂任务 → 自动判断是否值得固化 → 自动写入  |
| **渐进式加载** | 三级渐进式加载（元数据→全文→执行，分级较粗） | 三级渐进式加载 **Level 0/1/2 细化版**：Level 0（**3000 token** 概要）、Level 1（完整步骤）、Level 2（深度参考），按 token 量精确控制  |
| **GEPA 进化算法** | 优化靠外挂技能 | **内置离线进化引擎**（hermes-agent-self-evolution），遗传 + 帕累托提示词进化，技能越用越强  |
| **技能与记忆绑定** | Skill 与记忆系统弱关联 | **心跳时自动**：清理过时技能、合并重复技能、优化高频技能 → 系统级自我维护  |
| **生态标准** | 兼容 agentskills.io 标准 | 兼容 agentskills.io 标准，**可直接导入 OpenClaw 技能**，但 OpenClaw 不能导入 Hermes 进化型技能  |

### AI Coding 增强（4 大对比）
| 对比 | OpenClaw | Hermes |
|------|----------|--------|
| **项目记忆深度** | 能记住项目，但依赖 `.md` 文件，重启后可能需要重新加载 | **LanceDB 向量记忆 + 四层级加载**，**永久记住**架构、技术栈、命名习惯、错误历史、部署流程，**数月后回来继续写，完全不用重讲**  |
| **调试闭环成熟度** | 能调试，但有时会卡循环、重复犯错 | **自主反思机制**：自动记录错误模式、自动避免第二次踩坑、自动优化修复策略  |
| **自主规划能力** | 能做，但需要给大致方向、偶尔确认步骤 | 拿到需求 → **自动拆解任务、自动排期、自动分步执行、自动联调模块 → 几乎不用干预**  |
| **长期运行稳定性** | 长时间跑偶尔会丢上下文，技能多了变慢，偶尔需要重启 | 专为 **7×24 小时自主开发** 设计，内存管理更好，技能加载更快 → **长期不崩、不卡、不漂移**  |

### 任务并行
- **主 Agent 聊天和调度，同时派生出多个子 Agent 干活** 
- 子 Agent **一无所知**，主 Agent 只传：当前任务目标、精简版项目上下文、与任务直接相关的对话片段、权限与环境信息 
- 设计原因：
  1. **控制上下文长度，不爆 token** 
  2. **任务隔离，子 Agent 只专注干一件事** 
  3. **速度快**（无冗余信息）
  4. **安全，避免子 Agent 意外读取隐私历史** 

### 任务失败隔离
- 核心原则：重型任务永远子 Agent 干活，**主 Agent 永远不会挂** 
- 子 Agent 失败不影响主 Agent 和其他子 Agent，用户始终能与主 Agent 正常对话 

### 完整对比矩阵
| 维度 | OpenClaw | Hermes |
|------|----------|--------|
| 记忆结构 | 两级 | **四级**（标题→摘要→全文→深层参考） |
| 状态持久化 | 内存存储，重启丢失 | **JSON 持久记录**  |
| 心跳频率 | 固定间隔 | **动态智能调整**  |
| 主动工作 | 简单任务 | **管家级维护**  |
| 任务执行 | 单任务串行 | **多任务并行**  |
| 容错能力 | 主 Agent 可能受影响 | 子 Agent 失败，**主 Agent 永不挂**  |
