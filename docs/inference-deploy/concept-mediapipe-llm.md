# MediaPipe LLM 推理框架

> **类型**: concept
> **创建时间**: 2026-06-12
> **最后更新**: 2026-06-12
> **来源**: 

## 摘要

MediaPipe LLM 是 Google 推出的边缘推理框架，核心思路是将 LLM 推理抽象为**有向无环图（DAG）**，通过声明式编程让框架自动并行化和调度。最大护城河是**多模态原生支持**——实时视频/图像处理流水线与大语言模型无缝融合，特别适合 AR/视觉+对话场景。

## 详情

### 设计哲学——"把推理画成一张流程图"

MediaPipe 的设计思想来自 Google 在实时视频/图像处理流水线上的经验：

| 设计原则 | 含义 |
|---------|------|
| 计算图抽象 | 推理过程 = 一张有向无环图（DAG），能并行的地方自然并行了 |
| 模块化设计 | 每个计算节点独立可复用，像乐高一样搭积木 |
| 跨平台一致性 | 同一张图定义，Android/iOS/桌面跑出来一样 |
| 实时性优先 | 为流式处理和低延迟而生 |

**声明式 vs 命令式**：定义一张图（Input → Embedding → Layer0 → ... → Output），框架自己决定怎么并行、怎么调度。

**Calculator（计算节点）**：每个独立处理单元有三类端口——输入端口（Input Streams）、输出端口（Output Streams）、侧边输入（Side Packets）。生命周期：`Open() → Process() × N 次 → Close()`。

### 计算图抽象

LLM 推理图的节点类型：输入节点（Token 嵌入、位置编码）、计算节点（Transformer 层、注意力计算）、缓存节点（KV Cache 管理）、输出节点（Token 生成）。

**LLM 特有的图优化**：

1. **动态子图**：Prefill 阶段启用批量并行，Generation 阶段切换到增量计算，Early Exit 根据置信度动态跳过某些层
2. **时间戳对齐**：天然支持流水线并行，不同 token 在不同层同时计算
3. **背压控制**：下游慢了就自动减速上游——限制输入队列大小、自适应调整批处理大小、按优先级调度

**图编排性能效果**（32 层 Transformer，每层计算时间 t）：

| 策略 | 耗时 | 代价 |
|------|------|------|
| 串行执行 | 32t | 慢，但省内存 |
| 4 级流水线并行 | 8t + 3×延迟 | 快 4 倍 |
| 数据并行(batch=4) | 8t | 需要 4 倍内存 |

通过图分析，MediaPipe 能精确知道每个张量的生命周期，实测能**减少 40-60% 峰值内存使用** 。

### 设备端优化策略

**分层内存池**：L1 小张量池（<1KB）、L2 中等张量池（1KB-1MB）、L3 大张量池（>1MB）

**伙伴系统（Buddy System）减少碎片**：每次分裂产生一对大小相同的「伙伴」，释放时只有伙伴也空闲才能合并。

**内存压缩**：对生命周期 >100ms 的临时张量，用 LZ4 实时压缩（压缩比 2-3 倍，速度 >500MB/s）。

**GPU 委托**：三种后端可选——OpenGL ES（Android，最广泛兼容）、Metal/MPS（iOS/macOS，Apple 平台最优）、Vulkan（跨平台，最细粒度控制）。

**GPU 内存划分**：权重缓冲区 2GB（CPU 可见） + 激活缓冲区 512MB（GPU 专用） + 临时缓冲区 256MB（计算中间结果）

**自适应混合精度**：层敏感度 > 阈值 → FP16，层类型 == EMBEDDING → INT8，否则 → INT4

**量化 kernel 融合**：传统流程（反量化→计算→量化）→ 融合流程（INT8 直接算）

### 多模态支持——MediaPipe 的杀手锏

这是 MediaPipe 最大的护城河：实时视频/图像处理流水线与大语言模型无缝集成。

**统一流水线设计**：
```
摄像头 → 图像预处理 → 视觉编码器(ViT) ────┐
                                          ├→ 多模态融合 → LLM → 输出
文本输入 → 文本分词器 ─────────────────────┘
```

**异步流水线**：摄像头采集、预处理、ViT 编码、特征融合各阶段流水线满负荷运转

**自适应帧率控制**：场景复杂→10fps（算力留给理解），检测到运动→30fps（跟上变化），静态→5fps（省电）

**特征金字塔缓存**：4 级分辨率，不同级别不同更新频率（原始分辨率每帧更新，1/2 分辨率每 2 帧，1/4 分辨率每 4 帧，全局特征每 8 帧）

**跨模态注意力三层架构**：
- Early Layers：文本自注意力 | 视觉自注意力（各自独立）
- Middle Layers：轻量跨模态注意力（降采样后的廉价交互）
- Late Layers：完整跨模态注意力（只在关键层做重计算）

**实际应用案例**：实时视频理解（<100ms/查询）、AR 增强现实（60fps UI + 10fps AI）、文档理解 OCR+LLM（增量式理解）
