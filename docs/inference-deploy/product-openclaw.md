## 摘要
OpenClaw 是一个可持续运行的智能体任务系统，把远端入口、会话管理、工具执行和长期记忆缝合在一起，实现远程消息交互、智能执行、持续协作和可操作的系统能力。

## 详情

### 四大核心能力
| 编号 | 能力 | 说明 |
|------|------|------|
| 01 | **远端消息交互** | WhatsApp / Telegram / Slack / 飞书 / 微信 ...  |
| 02 | **智能执行** | 问答、总结、检索、生成、工具调用  |
| 03 | **持续协作** | Session、上下文、长期记忆、后续跟进  |
| 04 | **可操作** | 浏览器、系统工具、消息动作、设备能力  |

### 核心响应过程
主链路：**入口 → 接入 → 准备 → 主回路 → 回写 → 沉淀 → 继续/结束** 

三大特性：
- **可见**：过程里发生什么，渠道和 UI 能持续看到 
- **可恢复**：超时、重试、压缩上下文、模型切换都在同一条链里 
- **可继续**：这一轮做过的事，会变成下一轮还能接着用的状态 

### 关键构件分层
| 层级 | 构件 | 说明 | 扩展性 |
|------|------|------|--------|
| **会话控制层** | Gateway | 总线/鉴权/连接/事件 | **高**  |
| | Session | 上下文桶/路由落点/并发边界 | **低**  |
| **智能层** | Agent Loop | 一轮一轮推进的模型+工具执行回路 | **低**  |
| | Memory | 落在外部存储里的长期记忆 | **低**  |
| **行动能力层** | Skills | 通过 SKILL.md 告诉系统在什么场景怎么用工具 | **高**  |
| | ACP | 利用 ACPX 与其他 Agent 协作完成任务 | **高**  |

### Agent Loop 原理
```
Input → Plan → Action → Action Result(叠加) → Observe → Plan (循环)
```
- 在"补上上下文 → 判断下一步 → 执行动作 → 观察结果"之间循环 

### Memory 记忆机制
- 以 **Markdown 文件**形式存储，结构清晰、人人可读写 
- **每日记录**：`memory/YYYY-MM-DD.md`，记录当日原始日志 
- **长期记忆**：`MEMORY.md`，精选记忆库，仅在主会话加载 
- 记忆维护：每隔数日，利用心跳完成记忆提炼 
- 存储策略：**LanceDB 优先 > 本地文件降级** 

### 心跳（Heartbeat）机制
- 约 **20-40 分钟一次**轮询 
- 批量检查：邮件、日程、社交提及、天气 
- 巡检记录写入 `heartbeat-state.json` 
- 主动联络时机：收到重要邮件、日程不足 2 小时、距上次互动超过 **8 小时** 
- 保持静默场景：深夜时段 **23:00-08:00**、使用者忙碌、距上次巡检不足 **30 分钟** 
- 无需请示可主动完成：阅读整理记忆、检查 Git、更新文档、提交推送代码 

### Skill 技能体系
- **纯文本定义**：人人可写，易维护、可审计、可版本管理 
- **三级渐进式加载**：元数据加载 → 全文加载 → 执行加载 
- **多层级来源**：工作区技能、全局技能、内置技能、插件技能 
- **双触发模式**：自然语言描述、直接命令调用 
- **沙箱隔离与权限管控**：代码类 Skill 运行在沙箱中 

### AI Coding 行为准则
- **默认执行**：自己写的代码直接运行 
- **系统命令**：需要运行系统命令时会找用户确认 
- **权限**：当前登录用户权限 
- **文件读取范围**：指定工作目录中的所有文件 

### 核心提示词文件体系
| 文件 | 作用 |
|------|------|
| **AGENTS.md** | 工作空间规则指南  |
| **SOUL.md** | 安全规则和做人准则  |
| **IDENTITY.md** | 身份信息  |
| **USER.md** | 服务对象信息  |
| **TOOLS.md** | 本地工具配置笔记  |
| **HEARTBEAT.md** | 心跳任务清单  |
| **BOOTSTRAP.md** | 初始设定文件（首次运行时存在，完成后删除） |

### 模型应用形态演进
```
ChatGPT → GPTs → 低代码 → Coding Agent/CLI → OpenClaw
```
- OpenClaw 没有产生决定性跃迁，但把"**远程消息 IO + 会话 + 工具 + 回写 + 记忆**"的长时运行任务系统直观做给了用户看
