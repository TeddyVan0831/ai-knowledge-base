# Query 改写（查询改写）

> **类型**: concept
> **创建时间**: 2026-05-26
> **最后更新**: 2026-05-26
> **来源**: 

## 摘要

Query 改写是 RAG 系统中将口语化、模糊或多意图的用户查询转换为书面化、精确的检索语句的技术，是 RAG "检索-生成"流程中提升检索质量的关键环节。包含上下文依赖型、对比型、模糊指代型、多意图型和反问型五种改写类型。

## 详情

### 为什么需要 Query 改写

**问题**：用户提出的问题往往是口语化的、承接上下文的、模糊的，甚至是包含了情绪的。而知识库里的文本（切片/Chunks）通常是陈述性的、客观的。这种差别可能导致检索出的结果质量不及预期。而RAG 的核心在于"检索-生成"，如果第一步"检索"就走偏了，那么后续的"生成"质量也会降低 。

**解决**：需要"翻译官"将口语化查询转换为书面化、精确的检索语句 。

### 五种 Query 改写类型

#### 1. 上下文依赖型

**特征**：包含"还有"、"其他"等需要上下文理解的词汇 。

**示例** :
```
对话历史:
用户："我想了解上海迪士尼乐园的最新项目"
AI："推出了'疯狂动物城'主题园区"
当前查询："这个园区有什么游乐设施？"
改写结果："除了疯狂动物城主题园区，还有其他游乐设施吗？"
```

#### 2. 对比型

**特征**：包含"哪个"、"比较"、"更"等比较词汇 。

**示例** :
```
原始查询："哪个游玩的时间比较长，比较有趣"
改写结果："哪个游玩时间更长、更有趣：上海迪士尼乐园的疯狂动物城
          主题园区和蜘蛛侠主题园区？"
```

#### 3. 模糊指代型

**特征**：包含"它"、"他们"、"都"、"这个"等指代词 。

**示例** :
```
对话历史:
用户："我想了解上海迪士尼和香港迪士尼的烟花表演"
AI："都有精彩的烟花表演"
当前查询："都什么时候开始？"
改写结果："上海迪士尼乐园和香港迪士尼乐园的烟花表演都什么时候开始？"
```

#### 4. 多意图型

**特征**：包含多个独立问题，用"、"或"？"分隔 。

**示例** :
```
原始查询："门票多少钱？需要提前预约吗？停车费怎么收？"
分解结果：["门票多少钱？", "需要提前预约吗？", "停车费怎么收？"]
```

#### 5. 反问型

**特征**：包含"不会"、"难道"等反问语气 。

**示例** :
```
对话历史:
用户："我想预订下周六上海迪士尼乐园的门票"
AI："查询到下周六的门票已经售罄"
当前查询："这不会也要提前一个月预订吧？"
改写结果："迪士尼乐园门票是否需要提前一个月预订？"
```

### 自动意图识别

自动识别查询类型的指令模板 :
```
请分析用户查询，识别其属于以下哪种类型：
1. 上下文依赖型 - 包含"还有"、"其他"等
2. 对比型 - 包含"哪个"、"比较"、"更"等
3. 模糊指代型 - 包含"它"、"他们"、"都"、"这个"等
4. 多意图型 - 包含多个独立问题
5. 反问型 - 包含"不会"、"难道"等反问语气

说明：如果同时存在多意图型、模糊指代型，优先级为多意图型>模糊指代型
```

### 联网搜索场景中的 Query 改写

当 query 涉及的场景需要联网搜索后才能回答时，分为三步 :
1. 识别是否需要联网搜索
2. 为联网搜索改写查询
3. 生成搜索策略

**需要联网搜索的 8 类场景** :

| 类型       | 关键词特征          | 示例查询            |
| ---------- | ------------------- | ------------------- |
| **时效性**  | 最新、今天、现在、实时、当前 | "上海迪士尼乐园今天开放吗？" |
| **价格信息** | 多少钱、价格、费用、票价   | "下周六的门票多少钱？"    |
| **营业信息** | 营业时间、开放时间、闭园时间 | "迪士尼乐园现在开门吗？"   |
| **活动信息** | 活动、表演、演出、节日、庆典 | "最近有什么特别活动？"    |
| **天气信息** | 天气、下雨、温度       | "明天去迪士尼天气怎么样？"  |
| **交通信息** | 怎么去、交通、地铁、公交   | "从浦东机场怎么去迪士尼？"  |
| **预订信息** | 预订、预约、购票、订票    | "需要提前多久预订？"     |
| **实时状态** | 排队、拥挤、人流量      | "现在人多不多？"       |
