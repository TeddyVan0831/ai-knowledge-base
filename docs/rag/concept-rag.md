## 摘要

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的技术，通过实时检索相关文档或信息并将其作为上下文输入到生成模型中，从而提高生成结果的时效性和准确性。

## 详情

### 定义

RAG（Retrieval-Augmented Generation，检索增强生成）是一种结合信息检索和文本生成的技术。RAG通过实时检索相关文档或信息，并将其作为上下文输入到生成模型中，从而提高生成结果的时效性和准确性 。

**工作流程**：用户提问 → 检索相关文档 → 检索结果作为上下文输入 → 生成模型输出 。

### 三种大模型应用开发模式对比

| 模式       | 适用场景   | 解决问题            |
| ---------- | ---------- | ------------------- |
| **提示工程** | 没问清楚   | 优化问题表达，引导模型正确理解 |
| **RAG**  | 缺乏背景知识 | 引入外部知识库，增强回答准确性 |
| **微调**   | 能力不足   | 针对特定任务训练模型      |

### RAG 三阶段核心流程（Native RAG 三要素）

```
Step 1: 数据预处理（Indexing）
├─ 知识库构建：收集文档、网页、数据库等多源数据
├─ 文档分块：切分为适当大小的片段（chunks）
└─ 向量化：使用 Embedding 模型转换为向量，存入向量数据库

Step 2: 检索阶段（Retrieval）
├─ 查询处理：将用户问题转换为向量
├─ 相似度检索：在向量数据库中找到最相关的文本片段
└─ 重排序：对检索结果进行相关性排序，把最相关的片段输入生成阶段

Step 3: 生成阶段（Generation）
├─ 上下文组装：检索片段 + 用户问题
└─ 生成回答：LLM 基于增强上下文生成最终答案
```

来源: 

**Native RAG 三要素** :

| 要素             | 核心问题          | 关键工作            |
| ---------------- | ----------------- | ------------------- |
| **Indexing**   | 如何更好地把知识存起来   | 文档分块、向量化、索引构建   |
| **Retrieval**  | 如何在大量知识中找到有用的 | 相似度检索、重排序       |
| **Generation** | 如何结合检索知识生成答案  | 上下文组装、prompt 设计 |

### 核心优势

| 优势           | 说明               |
| -------------- | ------------------ |
| **解决知识时效性**  | 检索外部知识库，实时更新信息   |
| **减少模型幻觉**   | 引入外部知识，减少虚假内容    |
| **提升专业领域质量** | 结合垂直领域知识库，生成专业回答 |

来源: 

### RAG 即使面对无限上下文 LLM 仍有意义的原因

- **效率与成本**：LLM处理长上下文时计算资源消耗大，响应时间增加。RAG通过检索相关片段，减少输入长度 。
- **知识更新**：LLM的知识截止于训练数据，无法实时更新。RAG可以连接外部知识库，增强时效性 。
- **可解释性**：RAG的检索过程透明，用户可查看来源，增强信任。LLM的生成过程则较难追溯 。
- **定制化**：RAG可针对特定领域定制检索系统，提供更精准的结果，而LLM的通用性可能无法满足特定需求 。
- **数据隐私**：RAG允许在本地或私有数据源上检索，避免敏感数据上传云端，适合隐私要求高的场景 。
