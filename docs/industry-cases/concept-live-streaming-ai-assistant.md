# 直播间 AI 评论互动助手

> **类型**: concept
> **创建时间**: 2026-06-12
> **最后更新**: 2026-06-12
> **来源**: 

## 摘要

一种基于 Chrome 插件 + Tampermonkey 脚本 + Coze 工作流的直播间 AI 助手架构，能够自动抓取抖音直播间评论区的问题，通过 AI 判断是否值得回复，并自动以主播身份发布回复。核心思路是用浏览器插件"桥接"前端页面数据与后端 AI 工作流，将直播间的评论流和 AI 处理能力串联起来。

## 详情

### 架构设计

采用三步流程：

```
Tampermonkey 脚本抓取评论区 → 发送到 Coze 工作流 → 返回回复并自动发布
```



**三个核心能力**：
1. **抓取评论** — Tampermonkey 脚本从抖音直播间 DOM 中提取评论区内容
2. **调用 AI** — 脚本将评论发送给 Coze 工作流，工作流判断是否需要回复
3. **发布回复** — 脚本将 AI 生成的回复填入评论区并自动发送

### Chrome 插件生态

使用两个现成插件而非从零构建：
- **CSP 插件**：解除浏览器跨域安全限制，允许脚本调用 Coze API
- **Tampermonkey（篡改猴）**：在指定页面自动运行自定义脚本，读写 DOM、调用外部 API 

### Coze 工作流设计

工作流 `抖音_live` 结构极简：

```
开始节点（接收 question）→ LLM 节点（判断+回复）→ 结束节点（输出 JSON）
```

输出格式：
```json
{
  "need_reply": true,      // 是否需要回复
  "message": "亲，这款面膜..." // 回复内容
}
```



Prompt 要点：扮演人类年轻女性口吻回答产品相关问题，闲聊类不回复，回复尽可能短。

### 测试效果

| 场景 | 结果 |
|------|------|
| 不锈钢杯子直播间 | ✅ 成功回复"有 316 材质的" |
| 珠宝直播间（昵称打码） | ❌ 无法运行 |
| 衣服源头工厂直播间 | ✅ 成功回复 |
| 闲聊评论 | 大部分正确过滤 |



### 限制与扩展

**当前局限**：插件发布逻辑不稳定、昵称被隐藏时失效、API 调用频次限制（免费版 300 次/分钟）。

**扩展方向**：增加知识库（产品问答）、对接库存系统、更复杂的回复策略。
