## 摘要

一个以财务预算表格为切入点的 AI 产品案例，系统测试了通义千问 2.5、DeepSeek V3、GPT-4o 在表格数据识别（94.3%）、数据计算（93%-97%）和数据纠错（60%-75%）三方面的能力，并总结出一套标准化模型选型方法论：用**自己的业务场景数据**（而非通用榜单）从 Day One 开始做评测。

## 详情

### 预算表格的业务背景

财务预算表格包含成本预测、营收预测、金额及其占比/增长/分析。课程使用两个样例：To B 企业预算（产品线单一、人力成本占比高）和 To C 企业预算（200+ 行、推广渠道复杂）。预算的核心价值在于可控性、资源分配、横向对比和风险预警。

### 测试方法论

**核心原则**："没有单独的模型选型步骤。你选模型时怎么评测模型和你做了 AI 产品后怎么评测 AI 产品，是一回事。"

```
Day 1: 梳理场景 → 列出子场景 → 打标签 → 准备测试数据（每个子场景几十到几百条）
Day 2: 用同一套数据测试多个模型 → 统计正确率
Day 3: 基于测试结果选择模型 → 搭建应用层
持续: 用同一套数据评估产品上线后表现，计算"附加值"
```



**测试数据要求**：覆盖面广（每子场景几十上百个问题）、分层标签（三级）、明确评判标准、概率思维（每个问题运行 10 次统计正确率）、表述多样性（同一问题不同方式问）。

### 测试一：表格数据识别

共 35 个测试问题，覆盖产品单价、月度销售额、歧义问题等。

| 模型 | 平台 | 正确率 |
|------|------|--------|
| 通义千问 2.5 | Coze | **94.3%** |
| GPT-4o | Coze | 94.3% |
| DeepSeek V3 | Coze | 91.4% |
| DeepSeek V3 | 自家平台 | **65.7%** |



**关键发现**：DeepSeek 自家平台在处理 Excel 时存在显著缺陷——**偏好读取公式而非已计算出的数值**，导致同样模型在自家平台仅 65.7%，在 Coze 上达 91.4%。"这不是模型能力，是数据预处理方式不同。" 

**歧义理解差异**：通义千问 2.5 能正确理解"HR 人力成本"指人事岗位成本，其他模型理解为全公司人力成本。

### 测试二：数据计算

共 30 个计算问题（加权加和、上下半年对比、占比/增长率等）。

| 模型 | 平台 | 正确率 |
|------|------|--------|
| 通义千问 2.5 | Coze | **96.7%** |
| GPT-4o | Coze | 93.3% |
| DeepSeek V3 | Coze | 96.7% |



**Prompt 优化效果**：增加"可以输出方法或步骤"的引导后，模型输出中间推理过程（思维链），正确率显著提升。但出现了反直觉现象——简单计算（求和）反而答错，复杂计算（人均成本）稳定答对。

### 测试三：数据纠错

专用纠错工作流 `auto-fix` 设计：

```
Excel 上传 → 代码读取数据 → 并行双节点分析
                              ├── 节点 1：合规角度（人事/财务/法律）
                              └── 节点 2：数据错误角度（计算/逻辑）
                              ↓
                    合并输出（各找 5 个最严重的问题）
```



预设 5 个问题（4 个数据错误 + 1 个合规问题），模型平均找到 2-3 个，准确率 **60%-75%**（不稳定）。改进方向：第一轮不限量输出可疑问题 → 第二轮二次筛选判断（思维链思路）。

### Coze QA 工作流设计

```
开始节点 → 代码执行器（生成 Python 代码读取 Excel → 转为 JSON 纯文本）
         → LLM 节点（接收文本 + 用户问题 → 回答）
         → 结束节点
```



关键设计：用大模型自动生成 Python 代码解析 Excel，转为纯文本（JSON 格式），避免模型处理复杂表格格式。当前 Prompt 简单直接，未做倾向性引导，用于测试模型基础能力。

### 核心结论

| 维度 | 结论 |
|------|------|
| 数据识别 | 主流模型在 Coze 均达 ~94%，已可用 |
| 数据计算 | 93%-97%，通过思维链可进一步提升 |
| 数据纠错 | 60%-75%，需进一步优化 |
| DeepSeek 自家平台 | Excel 处理缺陷致仅 65.7%，不建议用于表格场景 |
| 时段影响 | 同一模型不同时段表现有差异，建议多次测试取平均 |
